# Architecture

## Overview

```
CLI (cli.py)                    Pipeline (steps.py)                FiftyOne App
────────────                    ───────────────────                ─────────────
yolo-scout data=... task=... ──▶ @pipeline decorator
                                  ├── validate()
                                  ├── prepare_run()
                                  ├── prepare_plugins()
                                  ├── run_load_dataset()    ◀── dataset/
                                  ├── run_embeddings()      ◀── embeddings/
                                  ├── run_quality_metrics() ◀── visualization/
                                  ├── run_thumbnails()      ◀── visualization/
                                  └── launch_app()          ───▶ Browser UI
```

YoloScout uses a pipeline architecture where each step is decorated with `@step`
for automatic logging and timing. The `@pipeline` decorator on the main CLI
function orchestrates all steps sequentially.

## Components

### CLI layer

!!! info "`cli.py`"

    The CLI parses `key=value` arguments from the command line. The `@pipeline` decorator wraps the main function to provide logging, timing, and error handling. No framework (Click, argparse) is used - arguments are parsed directly.

### Configuration

!!! info "`core/config.py` · `core/enums.py` · `cfg/default.yaml`"

    Configuration follows a three-level hierarchy with clear precedence:

    1. **Defaults** - loaded from `cfg/default.yaml`
    2. **Config file** - optional YAML file via `config=path.yaml`
    3. **CLI arguments** - highest priority, override everything

    `DatasetTask` and `EmbeddingsModel` enums provide type-safe task and model selection.

### Dataset loading

!!! info "`dataset/loader.py` · `dataset/parser.py` · `dataset/converter.py`"

    The loader discovers dataset splits, reads class names from `data.yaml`, and creates a persistent FiftyOne dataset. The parser handles all five YOLO annotation formats (detect, classify, segment, pose, obb). The converter transforms parsed annotations into FiftyOne label objects (Detection, Keypoint, Polyline).

    Two directory layouts are supported:

    - **Type-first**: `images/{split}/*` and `labels/{split}/*`
    - **Split-first**: `{split}/images/*` and `{split}/labels/*`

### Embeddings

!!! info "`embeddings/computer.py` · `embeddings/preprocessing.py`"

    Computes CLIP image embeddings and per-patch embeddings using one of four supported models. UMAP dimensionality reduction creates 2D visualizations. Patch preprocessing extracts annotation crops with optional background masking for segment and OBB tasks.

### Visualization

!!! info "`visualization/quality.py` · `visualization/iou.py` · `visualization/fiftyone_ops.py`"

    - **Quality metrics**: blurriness (Laplacian variance), brightness, entropy, aspect ratio
    - **IoU scores**: bounding box and polygon intersection-over-union via Shapely
    - **FiftyOne app**: launches the web UI with Ultralytics-style color schemes

### Pipeline orchestration

!!! info "`pipeline/steps.py` · `utils/decorators.py`"

    Each processing step is a function decorated with `@step`, which logs start/end times and handles errors. The pipeline runs steps in order:

    1. `validate()` - load and validate configuration
    2. `prepare_run()` - check cache, handle reload
    3. `prepare_plugins()` - install FiftyOne plugins
    4. `run_load_dataset()` - create or load FiftyOne dataset
    5. `run_embeddings()` - compute CLIP embeddings and UMAP
    6. `run_quality_metrics()` - compute image quality scores
    7. `run_thumbnails()` - generate image thumbnails
